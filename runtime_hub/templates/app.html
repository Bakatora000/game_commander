Déployer une instance

Déploiement rapide depuis le Hub avec les paramètres essentiels. Les réglages fins se font ensuite dans le Commander de l’instance.

Exemple : minecraft2. C’est le nom technique de l’instance : URL /minecraft2, dossier game-commander-minecraft2, services associés. Ce n’est pas l’utilisateur Linux ; le Hub réutilise l’utilisateur système par défaut du serveur.

Nom visible côté jeu quand le jeu le supporte. Ce n’est pas l’identifiant technique de l’instance.

Intégration Discord

Le bot token se configure dans /etc/game-commander/discord.json. Les paramètres ci-dessous sont sauvegardés dans ce même fichier.

Serveur Discord

Dans Discord : clic droit sur l'icône de ton serveur → Copier l'identifiant du serveur. Nécessite le mode développeur (Paramètres utilisateur → Avancé).

Canal Discord utilisé pour les notifications des instances sans canal dédié. Tape # pour voir les canaux disponibles, ou colle un ID directement.

Normalement inutile : une catégorie par jeu est créée automatiquement (valheim, enshrouded…). Ce champ sert de repli uniquement si le jeu n'est pas identifiable.

Canaux d'instances

Un canal Discord par instance, créé automatiquement au déploiement si l'ID du serveur est configuré ci-dessus.
